# 前端说明

## 技术栈

- **Alpine.js 3.x**（CDN 引入，无构建步骤）
- 纯 HTML + CSS，单文件 `static/index.html`
- 浅色系配色，支持桌面和手机浏览器

## 设计约定

- 每队**固定 3 名武将**（1 主将 + 2 副将），不提供添加/删除按钮
- 每名武将**固定 3 个战法槽**，不提供添加/删除按钮
- 所有数值（武力/智力/统率/速度/兵力/战法参数）均为可编辑输入框
- 科技配置每队各一组（攻击/谋略/防御科技 + 九宫图 + 八卦阵）

## 颜色变量

```css
--bg: #f4f1eb;
--surface: #ffffff;
--surface2: #faf8f4;
--border: #e0d8cc;
--accent-a: #2563eb;   /* 我方（蓝色） */
--accent-b: #dc2626;   /* 敌方（红色） */
--green: #16a34a;
--gold: #b45309;
```

## 默认阵容

| 队伍 | 主将 | 副将1 | 副将2 |
|------|------|-------|-------|
| A（我方） | 关羽（骑兵S） | 张飞（枪兵A） | 诸葛亮（弓兵S） |
| B（敌方） | 吕布（骑兵S） | 张辽（枪兵A） | 貂蝉（弓兵A） |

## 主要交互

- **单次模拟**：返回详细战斗日志，可折叠展开
- **批量模拟**：可配置次数（默认100），返回胜率统计
- 结果面板：胜/负/平局 banner + 各武将剩余兵力 + 损失百分比

## 注意事项

- 前端直接调用 `/api/simulate`，无需跨域配置（同域部署）
- Alpine.js 的 `x-model.number` 用于数字输入绑定
- 战斗日志按 engagement（局）分组展示
